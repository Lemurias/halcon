# HALCON-Core

This project is part of [HALCON](https://gitlab.com/hawk-dsp) and contains the ***core*** used in [examples](https://gitlab.com/hawk-dsp/examples).

### Launch Docker container

The container's description is located in the [tools](https://gitlab.com/hawk-dsp/tools) repository, which is included as a submodule. To *build* the container image, execute the following commands:

```bash
cd {CORE}/tools/docker
sh build.sh
```

To *run* the container image, move to the main folder (`{CORE}`) and execute the following script:

```bash
cd {CORE}
sh tools/docker/run.sh
```

### Build Core

In the Docker container terminal, run the following commands:

```bash
cd {CORE}/build
./build.py -h
./build.py -d -f -p
```

If everything has proceeded as expected, the static library will be exported to the directory `core/release`. The *release hash code*  is saved in `core/release/release.hash`.

### Publish a release

To commit to the [release](https://gitlab.com/hawk-dsp/release) repository, including the *hash* in the commit message, exit the Docker container and execute:

```bash
cd {CORE}/release
sh push.sh
```

### Debug simulator

Inside the `debug/` directory there is a *test simulator* that is used to develop and debug the Core. Each simulator includes a `main.py` script used for compiling and running the simulations, among other tasks. To list the available options for this script, execute:

```bash
cd {CORE}/debug
./main.py -h
```

To compile the Full Core (modeling positive and negative edges of clocks) run the following commands:

```bash
./main.py -f
```

Once the compilation is finished, you can run it using the following command:

```bash
./main.py -r -t
```

### Documentation

The documentation is powered by ``. To generate the project documentation in local mode, execute the following command *inside* the Docker container:

```bash
cd {CORE}
doxygen ./docs/doxygen.conf
```

To view the documentation open `./docs/html/index.html` with your web browser (*outside* the Docker container).
